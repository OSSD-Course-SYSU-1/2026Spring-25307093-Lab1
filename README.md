## 1. 项目概述
- **项目名称**：基于HarmonyOS的简易计步器应用
- **开发平台**：HarmonyOS（DevEco Studio）
- **开发语言**：ArkTS
- **项目简介**：
  本项目是一个基于HarmonyOS平台开发的轻量级计步器应用，通过调用设备传感器API实现实时步数统计，支持显示当前步数、消耗卡路里、运动时长与目标完成度。项目采用组件化开发思想，将数据采集、逻辑计算与界面展示分离，实现了低耦合、易维护的代码结构，适合作为鸿蒙传感器开发与状态管理的入门实践项目。

## 3. 核心模块详解
### 3.1 传感器数据采集模块
- **功能**：调用鸿蒙设备传感器API，实时监听加速度传感器数据，通过算法识别用户行走动作并统计有效步数。
- **实现方式**：使用`@ohos.sensor`模块注册传感器监听，通过滤波算法过滤无效震动数据，仅识别符合行走特征的加速度变化。

### 3.2 数据处理与计算模块
- **功能**：基于统计的步数数据，计算用户消耗的卡路里、预估运动时长与目标完成百分比。
- **实现方式**：根据预设的体重、步距等参数，通过公式换算卡路里消耗；根据平均步频估算运动时长，实现数据的实时更新。

### 3.3 界面展示模块
- **功能**：使用ArkTS基础组件构建UI，实时展示步数、卡路里、进度条等数据，实现数据与界面的双向绑定。
- **实现方式**：通过`@State`装饰器管理状态变量，结合`Text`、`Progress`、`Button`等组件，实现数据更新时界面的自动刷新。

## 4. 数据模型详解
项目定义了`StepData`数据模型，用于封装计步相关的核心数据：
```typescript
class StepData {
  stepCount: number;    // 当前步数
  calories: number;     // 消耗卡路里
  duration: number;     // 运动时长（分钟）
  targetSteps: number;  // 每日目标步数
  progress: number;     // 目标完成进度（0-100）
}
5. 工具类详解
项目封装了StepCalculator工具类，提供通用的数据计算方法：
• calculateCalories(steps: number): number：根据步数计算消耗的卡路里。
• calculateDuration(steps: number): number：根据步数估算运动时长。
• calculateProgress(steps: number, target: number): number：计算目标完成进度。
工具类将通用计算逻辑封装，避免了代码冗余，提高了代码的可复用性与可维护性。
6. 常量配置详解
项目通过Constants.ets文件统一管理全局常量，便于修改与维护：
export const Constants = {
  DEFAULT_WEIGHT: 60,    // 默认体重（kg）
  DEFAULT_STEP_LENGTH: 0.75, // 默认步距（m）
  DEFAULT_TARGET_STEPS: 8000, // 默认每日目标步数
  CALORIES_PER_STEP: 0.04,  // 每步消耗的卡路里
};
统一配置常量的方式，避免了 “魔法数字” 的出现，提高了代码的可读性与可维护性。
7. 资源文件详解
项目的资源文件统一存放在resources目录下，包含：
• media/：存放应用图标、进度条背景等图片资源。
• string.json：存放应用名称、提示文字等字符串资源，支持多语言配置。
通过资源文件的统一管理，实现了 UI 与数据的分离，便于后续的维护与扩展。
8. 功能流程分析
1. 初始化：应用启动时，读取传感器权限，初始化传感器监听与状态变量。
2. 数据采集：传感器监听实时获取加速度数据，通过算法识别有效步数并更新数据模型。
3. 数据计算：工具类根据更新的步数数据，计算卡路里、时长与进度等衍生数据。
4. 界面更新：数据模型更新后，界面组件自动刷新，展示最新的计步数据。
5. 用户交互：用户可通过界面设置目标步数、查看历史数据，应用响应交互并更新状态。
